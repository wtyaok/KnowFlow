# KnowFlow 项目二期优化任务（UI + 业务增强）

你已经完成了 KnowFlow 项目的一期版本，现在需要在现有代码基础上进行“企业级增强优化”，目标是：

👉 让系统更像真实企业内部使用的后台系统
👉 UI 更精致、更统一
👉 业务逻辑更完整一点，但不能复杂化
👉 不新增技术栈，不重构架构

---

# 一、整体优化目标（非常重要）

当前系统存在的问题：
- 页面略显简单
- 信息密度不够
- 企业后台“真实感”不足
- 模块之间联动弱
- UI组件缺乏统一设计规范

本次优化目标：

✔ 提升整体 UI 视觉层级（更企业后台风格）
✔ 增加“轻业务逻辑深度”
✔ 增强模块之间的数据关联
✔ 优化交互细节（loading / 状态 / 空状态）
✔ 让系统看起来像“真实公司内部系统”

---

# 二、UI整体升级要求（重点）

请统一调整整个系统 UI 风格：

## 1. 布局升级
- 左侧 sidebar 更精致（增加图标 + 分组标题）
- 顶部 header 增加：
  - 用户信息 dropdown
  - 系统通知 icon（假数据即可）
- 内容区域增加：
  - card 阴影层级
  - 更合理的 padding（整体更“松”一点）

---

## 2. 视觉风格统一

必须做到：

- 更强的卡片化 UI
- 统一 border-radius（8px 或 10px）
- 统一浅色阴影（不要太重）
- 主色保持蓝色体系，但更柔和
- 表格增加 hover 状态
- 按钮分 primary / secondary / danger 层级

---

## 3. 信息密度优化

当前问题是信息太“空”，请：

- 表格增加更多辅助字段
- card 中增加 secondary text
- 列表增加“状态 + 时间 + 来源”组合信息
- AI 对话增加结构层级（不要只有气泡）

---

# 三、业务逻辑增强（轻量但关键）

注意：不是增加复杂功能，而是“让系统更像真的”

---

## 1. 知识库模块增强

增加：

### （1）文档分类系统（轻量）
- 分类：HR / 技术 / 运营 / 产品
- 上传时可选择分类
- 列表支持按分类 filter

---

### （2）文档状态流转增强
文档状态改为：

- uploading
- parsing（新增）
- active
- failed

模拟：
- 上传后 → uploading
- 1秒后 → parsing
- 再1秒 → active

👉 体现“企业文档解析流程”

---

### （3）文档详情增强
增加：
- 文档元信息（上传人 / 时间 / 分类）
- “解析结果区域”（mock结构化内容）

---

## 2. AI对话模块增强（重点升级）

当前只是聊天 UI，需要增强“企业知识问答感”

### （1）回答结构升级

AI 回复必须拆成三块：

- 结论（summary）
- 依据（reasoning 简化版）
- 引用来源（sources）

---

### （2）引用来源升级

每条引用卡片必须包含：

- 文档标题
- 章节/片段
- 相似度（mock百分比，例如 87%）
- 标签（HR / 技术等）

---

### （3）增加“多来源回答”

每次 AI 回复必须至少 2 条引用来源（mock数据即可）

---

## 3. 工单系统增强（重点）

### （1）工单流转更真实

状态流：

pending → processing → resolved → closed

增加规则：

- pending：只能管理员处理
- processing：可以转 resolved / closed
- resolved：允许 reopen（回到 processing）

---

### （2）工单详情页增强

增加：

- 操作日志（mock timeline）
- 处理记录（谁在什么时候处理）
- 当前状态说明卡片

---

### （3）工单优先级细化

增加：

- low
- medium
- high
- urgent

UI上用不同颜色 badge 表示

---

## 4. Dashboard（轻量新增页面 ⭐）

新增一个“工作台 Dashboard”（不需要复杂逻辑）

包含：

- 文档总数
- AI问答次数
- 工单总数
- 待处理工单数

+ 一个简单趋势图（mock数据即可）

👉 用来增强“企业系统真实感”

---
